# Package Lifecycle: Offboarding

Use this checklist when deprecating, transferring, or removing a package from Big Bang.

- [ ] 1. Confirm governance path: Coordinate with BBTOC and package owners on deprecation/removal scope and timeline.
- [ ] 2. Coordinate security and stakeholders: Notify Cyber and other affected stakeholders of planned changes.
- [ ] 3. Remove technical integration artifacts: Remove or disable package chart wiring, values, tests, and CI jobs as needed.
- [ ] 4. Update documentation: Remove or mark package docs as deprecated and document migration guidance for users.
- [ ] 5. Communicate release impact: Capture user-facing impact in release notes and related communications.

When package ownership changes instead of full removal, keep the package integration docs current and update ownership expectations in [Package Owner Requirements & Overview](../package-integration/ownership.md).
