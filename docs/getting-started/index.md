# Getting Started

Start here if you are new to Big Bang or planning your first deployment.

## Recommended Path

1. Read [What is Big Bang?](../what.md) to understand Big Bang's scope and operating model.
2. Review [Prerequisites](prerequisites.md) before planning a cluster or deployment environment.
3. Use the [Quick Start Demo](../installation/environments/quick-start.md) for a hands-on demo deployment.
4. Use the [Customer Template](https://repo1.dso.mil/big-bang/customers/template) as the starting point for a real environment.
5. Review [Installation](../installation/) for environment-specific deployment guidance.

## Common Starting Points

- **[Prerequisites](prerequisites.md)**: Cluster, network, storage, and access requirements.
- **[Quick Start](../installation/environments/quick-start.md)**: Demo deployment path for learning and evaluation.
- **[First Deployment](first-deployment.md)**: Production-oriented starting point using the customer template.
- **[FAQ](faq.md)**: Common questions for new users.
- **[Concepts](../concepts/)**: Architecture, GitOps workflow, package management, and security model.
- **[Packages](../packages/)**: Core, add-on, and community package documentation.
